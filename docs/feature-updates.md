# 주요 기능 및 API 업데이트 (Feature Updates - Server)

이 문서는 `server` 모듈의 버전별 주요 기능 추가 및 API 변경 사항을 기록합니다.

---

## 전북대 수강신청 도우미 서버 정식 릴리스 (v1.0.0)

모든 핵심 비즈니스 로직과 알림 파이프라인이 탑재된 첫 번째 정식 버전입니다.

### 1. 전북대학교 오아시스 시스템 연동 및 크롤링 엔진 🕷️

- **실시간 강의 수집**: Jsoup을 이용한 전북대학교 오아시스 REST API 연동 및 주기적 강의 갱신 스케줄러(Quartz/Spring Scheduler) 구현.
- **크롤링 타겟 관리**: 하드코딩 탈피. 관리자가 DB에 저장된 목표 년도/학기(`CrawlerSetting`)를 자유롭게 제어 가능하도록 구성하여 무중단 학기 전환 구현.
- **안정성 메커니즘**: 외부 API의 느린 응답에 대비한 타임아웃 핸들링 및 실패 시 재시도 로직 도입, 명확한 일반 런타임 에러 처리(`CRAWLER_CONNECTION_ERROR`).

### 2. 고성능 정밀 강의 검색 시스템 🔍

- **동적 쿼리 처리**: `Querydsl`을 활용해 캠퍼스, 이수 구분, 대상 학년, 검색 키워드(교수명, 강의명) 등을 조합한 복잡한 조건 필터링 성능 제고.
- **백엔드 통합 찜 필터**: 수천 개의 강의 데이터에서 자신이 찜(관심 등록)한 과목만 정확하게 조회하는 서브쿼리 최적화 구축.
- **무한 스크롤 페이징 지원**: 성능 오버헤드를 막기 위해 카운트 쿼리를 배제한 `Slice` 인터페이스 기반의 빠른 결과 반환.
- **DTO 현지화 및 최적화**: 사용하지 않는 필드 제거 및 열거형(Enum) 응답(예: "전공필수") 등의 현지화 텍스트 직접 지원으로 프론트엔드 파싱 부하 경감.

### 3. 멀티 채널 푸시 알림 비동기 브로드캐스트 (Notification System) 🚀

- **이메일(Mailing) 발송**: Spring Mail을 활용해 전북대 학생들에게 친숙한 포맷의 여석 안내 메일 발송망.
- **브라우저/앱 푸시 (PWA)**: Firebase Cloud Messaging(FCM) 서버 연동을 통해 모바일/데스크탑 환경으로 Web Push 즉시 전달.
- **디스코드 채널 & DM 알림**: Discord OAuth 연동 및 봇/웹훅을 이용한 가장 빠른 실시간 서드파티 알림(게임 및 대기 중 가장 접근성 높은 매체) 지원.

### 4. 대학생 필수 인증과 안전한 구독 생태계 🔔

- **OAuth2.0 로그인 통합**: 카카오, 네이버, 구글 등 주요 플랫폼 소셜 로그인 지원 및 안전한 JWT Access/Refresh 토큰 인프라.
- **단계별 사용자 온보딩**: 최초 가입 유저의 DB 플래그(`onboardingCompleted`)를 기반으로 회원가입 직후 알림 수신용 정보(이메일, 디스코드 등)를 등록하도록 유도하는 체계.
- **빈자리 모니터링 구독 정책**: 수강신청 기간 여석 확인을 위해 강좌를 "구독"하는 도메인 및 중복 구독 방지 락 체계 설계.
- **구독 검증 가드**: "현재 시스템이 추적 중인 학기의 과목인지"를 인스턴스 메서드로 검증(`isMatchingTarget()`)해 엉뚱한 이전 학기 알림 구독 시도를 서버 레벨에서 원천 차단.

### 5. 어드민 전용 제어 및 통계 대시보드 API 📊

- **상세 데이터 시각화 API 연동**: 시스템의 트래픽(접속자 수, 알림 발송 건수), 에러율 등을 프론트 차트로 그리기 위한 실시간 시스템 현황 집계(통계) 엔드포인트 세팅.
- **시스템 활동 로그 기록 (Audit)**: 크롤링의 성공/실패 여부와 어떤 과목에 알림이 발송되었는지 타임라인을 기록하여 관리자에게 추적 데이터 API 제공.
- **직관적 학사 일정 관리 채널 (`/api/v1/admin/schedules`)**: 플라이웨이(Flyway) 기반 마이그레이션이 적용된 `Schedule` 엔티티를 CRUD 방식으로 유연하게 조작. (사용자 호출 시엔 지난 일정 자동 제외 및 D-Day 직렬화 배임 제공)
- **수동 크롤링 액션**: 어드민 전용 엔드포인트(`POST /api/v1/admin/courses/crawl/target`)를 통한 다이렉트 트리거 제공.

### 6. 아키텍처 및 코드 품질 🛠️

- **클린 코드 기반 설계**: SOLID 원칙 수용, 코드 레벨 Indentation 조절 및 중요 비즈니스 핵심부 Javadoc 한글 주석 작업.
- **분리 독립형 테스트 스위트 (WebMvcTest)**: 시큐리티 인증 객체나 JPA 메타모델 빈 로딩 의존성을 완벽하게 격리(`@MockitoBean`, `excludeFilters`)시킨 모듈형 컨트롤러 유닛 테스트 환경 확립.

### 7. 데이터베이스 형상 관리 및 도메인 최적화 (v1.1.0) 🛠️

- **마이그레이션 통합 및 최적화**: 분산되어 있던 `Flyway` 마이그레이션 파일(V3~V7)을 단일 파일(`V3__consolidate_schema_updates.sql`)로 통합하여 초기 배포 및 DB 초기화 리소스를 대폭 절감.
- **`course_schedules` 스키마 튜닝**: 불필요한 기간(`period`) 컬럼을 제거하고 인덱스 및 제약 조건을 재설계하여 크롤링 데이터 적재 성능 최적화.
- **일정(Schedule) 도메인 모델 단순화**: 사용자 혼선을 줄이기 위해 모호했던 `target` 필드를 과감히 삭제하고, 모든 일정을 `startDate`/`endDate` 및 `startTime`/`endTime` 체계로 일원화하여 표현력 강화.
- **테스트 정합성 확보**: 도메인 필드 변경에 맞춰 `ScheduleServiceTest`, `AdminScheduleControllerTest` 등 백엔드 전반의 테스트 스위트를 업데이트하여 변경 취약성 방어.
