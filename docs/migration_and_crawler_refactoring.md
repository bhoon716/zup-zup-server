# 데이터베이스 마이그레이션 및 크롤러 설정 리팩토링 (2026-05-07)

## 1. 개요
데이터베이스 초기화 과정에서 발생하던 Flyway 체크섬 불일치 오류를 해결하고, H2 데이터베이스 환경에서의 호환성을 확보하기 위해 전체 마이그레이션 스크립트 및 크롤러 관련 로직을 리팩토링했습니다.

## 2. 데이터베이스 마이그레이션 (Flyway)
H2 데이터베이스는 MySQL 전용 구문 및 복잡한 프로시저 제어를 완벽히 지원하지 않으므로, 모든 환경에서 안정적으로 동작하도록 스크립트를 최적화했습니다.

### 주요 변경 사항
- **V1__initial_schema.sql**
  - 테이블 생성 순서 재배치: 외래 키 제약 조건 오류 방지를 위해 부모 테이블을 자식 테이블보다 먼저 생성하도록 순서를 조정했습니다.
  - MySQL 전용 구문 제거: `ENGINE=InnoDB`, `CHARSET`, `COLLATE` 등 H2에서 오류를 유발하는 구문을 제거했습니다.
  - 기본 데이터 추가: `crawler_settings` 테이블에 초기 크롤링 타겟 데이터를 삽입하는 구문을 추가했습니다.
- **V2, V3, V5**
  - 프로시저(Procedure) 단순화: MySQL 전용 `DELIMITER` 및 `PROCEDURE` 블록을 제거하고, 표준 `ALTER TABLE` 구문으로 교체했습니다.
- **가독성 개선**
  - 모든 영문 주석을 한글로 변경했습니다. (예: `Table structure` -> `테이블 구조`)

## 3. 크롤러 타겟 설정 자동화
시스템 초기 구동 시 데이터베이스에 크롤링 설정이 없을 경우 발생하는 오류를 방지하기 위해 자동 초기화 로직을 도입했습니다.

### 주요 변경 사항
- **설정 파일(YAML) 연동**
  - `application.yml` 및 `application-prod.yml`에 `jbnu.crawler.default-year`와 `jbnu.crawler.default-semester` 옵션을 추가했습니다.
- **CourseCrawlerTargetService**
  - `getOrInitSetting()`: 데이터베이스 조회 결과가 없을 경우, 설정 파일의 기본값을 읽어 자동으로 초기 레코드를 생성하고 반환합니다.
  - 트랜잭션 보완: 초기화 과정에서 발생하는 쓰기 작업을 위해 관련 메서드에 `@Transactional`을 적용했습니다.

## 4. 향후 주의 사항
- `application.yml` 파일은 `.gitignore`에 포함되어 있어 Git 추적에서 제외됩니다. 새로운 환경 구축 시 해당 파일에 크롤링 기본 설정값이 포함되어 있는지 확인이 필요합니다.
- 마이그레이션 파일 수정으로 인해 기존 데이터베이스의 체크섬 오류가 발생할 수 있습니다. 개발 환경에서는 `application.yml`의 `flyway.validate-on-migrate: false` 설정을 활용하거나 DB 초기화를 권장합니다.
